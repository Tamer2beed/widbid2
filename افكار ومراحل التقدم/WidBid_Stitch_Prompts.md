## 📱 الشاشة 1: تسجيل الدخول (login.html)
**الغرض:** نقطة الدخول للمنصة — زائر أو عضو مسجل

### التخطيط العام:
- خلفية الشاشة: تدرج قطري من PRIMARY (#2C3E7A) في الأعلى إلى SECONDARY (#4A90D9) في الأسفل
- البطاقة المركزية: BG_CARD، border-radius 20px، ظل LG، تأخذ 90% من عرض الشاشة
- شعار WIDBID في الأعلى: نص أبيض، FONT_SIZE_XXL، FONT_WEIGHT_BOLD، مع خط تحته بلون ACCENT

### تبويبات الدخول (Tabs):
- 3 تبويبات متساوية العرض: "زائر" | "عضو" | "مسجل"
- التبويب النشط: خلفية PRIMARY، نص أبيض
- التبويبات غير النشطة: شفافة، نص TEXT_MUTED
- خط فاصل أسفل التبويبات بلون BORDER

### محتوى تبويب "زائر":
1. **حقل الاسم المستعار:**
   - ارتفاع: 52px
   - أيقونة شخص على اليمين (RTL)
   - placeholder: "اسم مستعار..."
   - border-radius: BORDER_RADIUS_SM
   - حد: 1px solid BORDER
   - عند التركيز: حد 2px لون SECONDARY

2. **اختيار الدولة:**
   - صف أفقي: علم دائري (32px) + اسم الدولة + سهم للأسفل
   - ارتفاع: 52px
   - نفس تصميم حقل الإدخال

3. **اختيار الأفاتار:**
   - عنوان صغير: "اختر صورتك"
   - شبكة 6×9 من الصور الدائرية (44px لكل واحدة)
   - الصورة المختارة: حد 3px بلون ACCENT + تكبير طفيف
   - قابلة للتمرير عمودياً

4. **زر دخول:**
   - Primary Button كامل العرض
   - نص: "دخول كزائر"

### محتوى تبويب "عضو" و"مسجل":
1. حقل اسم المستخدم (أيقونة شخص)
2. حقل كلمة المرور (أيقونة قفل + زر إظهار/إخفاء)
3. صف: checkbox "تذكرني" + رابط "نسيت كلمة المرور؟"
4. زر "دخول"
5. فاصل نصي "أو"
6. زر "إنشاء حساب جديد" (outlined style, لون SECONDARY)

## 📱 الشاشة 2: قائمة الغرف (rooms.html)
**الغرض:** استعراض الغرف حسب الدولة واختيار الغرفة المطلوبة

### الهيدر:
- خلفية: PRIMARY (#2C3E7A)
- ارتفاع: HEADER_HEIGHT (56px)
- شعار WIDBID: أبيض، مركز، FONT_SIZE_XL
- لا يوجد زر رجوع

### شريط التنقل السفلي:
- خلفية: BG_CARD (#FFFFFF)
- ارتفاع: BOTTOM_BAR_HEIGHT (60px)
- ظل علوي: 0 -2px 8px rgba(0,0,0,0.08)
- 4 أزرار متساوية:
  - 🏠 الغرف (نشط: لون PRIMARY، غير نشط: TEXT_MUTED)
  - ❤️ المفضلة
  - 🔍 بحث
  - ··· المزيد

### القائمة الرئيسية (قائمة الدول):
- كل عنصر: صف أفقي RTL، ارتفاع 68px
- تصميم العنصر:
  - **اليمين:** علم دائري بحجم 48×48px مع حد لطيف
  - **الوسط:** اسم الدولة (FONT_SIZE_MD، FONT_WEIGHT_BOLD) + صف ثانٍ: أيقونة 👥 + عدد مستخدمين + مسافة + أيقونة 💬 + عدد غرف
  - **اليسار:** نجمة ⭐ (ذهبية إذا مفضلة، رمادية إذا لا)
- فاصل رفيع بين العناصر: 1px BORDER
- أول عنصر: "الغرف المميزة ⭐" بخلفية ACCENT خفيفة جداً (10%)

### شريط الإجماليات (أسفل القائمة، فوق شريط التنقل):
- خلفية: BG_LIGHT
- حد علوي: 1px BORDER
- صفان: إجمالي المستخدمين | إجمالي الغرف
- نص: TEXT_MUTED، FONT_SIZE_SM

### شاشة البحث (عند الضغط على بحث):
- حقل بحث واسع في الأعلى مع أيقونة 🔍 وزر × للإلغاء
- placeholder: "ادخل جزء من اسم الغرفة أو رقمها"
- نتائج البحث: نفس تصميم قائمة الغرف

## 📱 الشاشة 3: داخل الغرفة — Guest View (chat.html)
**الغرض:** الواجهة الأساسية للدردشة — القاعدة التي تبنى فوقها كل الرتب

### الهيدر:
- خلفية: تتغير حسب ثيم الغرفة (افتراضي: #8B5CF6 بنفسجي)
- ارتفاع: HEADER_HEIGHT + padding إضافي = 64px
- **يمين:** زر مايك [🔊 Mic Free] — بطاقة بيضاء صغيرة، border-radius 8px، نص أسود FONT_SIZE_SM
- **وسط:** اسم الغرفة، نص أبيض، FONT_WEIGHT_BOLD
- **يسار:** 3 أيقونات بيضاء: 👥 (الأعضاء) | 💬 (الخاص) | ≡ (القائمة)

### منطقة المحتوى الرئيسية:
**عند إغلاق قائمة الأعضاء (الافتراضي):**
- عرض كامل للشات
- خلفية: لون الثيم مع شفافية 15% + نقش شفاف (SVG pattern للحلوى/الزهور/إلخ)

**عند فتح قائمة الأعضاء:**
- تقسيم: 35% للأعضاء | 65% للشات
- الانتقال: animation ناعم 0.25s من اليمين

**قائمة الأعضاء:**
- خلفية: BG_CARD
- كل عضو: صف ارتفاع 56px
  - أفاتار 36px (دائري، حد 2px بلون رتبته)
  - اسم ملوّن بلون رتبته، FONT_WEIGHT_MED
  - أيقونة 🎤 خضراء تومض عند تحدثه
- الترتيب: الأعلى رتبة أعلاه

**بانر الترحيب (رسالة نظام):**
- بطاقة بيضاء، border-radius 12px، ظل MD
- نص متوسط، لون TEXT_DARK
- هامش 16px من الجانبين
- يظهر أعلى منطقة الشات مباشرةً

**فقاعات الرسائل:**
- أفاتار 36px على اليمين (RTL) + محتوى على اليسار
- اسم المرسل: ملوّن بلون رتبته، FONT_SIZE_XS، FONT_WEIGHT_BOLD
- وقت الرسالة: TEXT_MUTED، FONT_SIZE_XS، بجانب الاسم
- نص الرسالة: FONT_SIZE_MD، TEXT_DARK
- الفقاعة: BG_CARD، border-radius 4px 12px 12px 12px، ظل SM
- مسافة بين الرسائل: SPACING_SM

### شريط الأدوات الأول (أساسي):
- خلفية: لون الثيم (أغمق بـ 20%)
- ارتفاع: 52px
- من اليمين إلى اليسار:
  - 🎤 (32px) — مايك
  - 💬··· (32px) — رسائل/مميزات
  - [حقل نصي: "اكتب رسالة..."] — مرن، خلفية بيضاء شفافة، border-radius 24px
  - ⌨️ (28px) — تبديل لوحة المفاتيح
  - 😊 (28px) — إيموجي
  - 📤 (28px) — إرسال/صورة

### شريط الأدوات الثاني (سريع):
- خلفية: نفس الهيدر
- ارتفاع: 40px
- أيقونات متساوية المسافات: ⏰ | 😐 | 🖐️ | ❤️ | 💡
- حجم كل أيقونة: 24px

### لوحة الإيموجي (تنبثق من الأسفل):
- ارتفاع: 280px
- تبويبات: ❤️ | 😊 | 🎉 | 🌿
- شبكة 8 أعمدة من الإيموجيات 32px
- قابلة للتمرير عمودياً

### لوحة إرسال صورة/كاميرا (تنبثق):
- بطاقتان جانبيتان:
  - "إرسال صورة" [أيقونة صورة خضراء + كاميرا زرقاء]
  - "كاميرا" [أيقونة كاميرا فيديو حمراء]

### القائمة الجانبية ≡ (تنبثق من اليسار RTL - من اليمين):
- عرض: 220px
- خلفية: BG_CARD
- ظل LG
- قائمة عناصر (ارتفاع كل عنصر: 52px):
  - 🔵 الحالة (مع السهم ← للقائمة الفرعية)
  - ⚙️ الإعدادات
  - ⭐ إضافة للمفضلة / حذف من المفضلة
  - 🗑️ مسح النص (نص أحمر — يظهر للمشرفين فأعلى)
  - 🚨 تبليغ
  - 🚪 خروج (نص DANGER)

### قائمة الحالات الـ7 (تنبثق عند الضغط على "الحالة"):
- قائمة منسدلة من اليمين
- خلفية داكنة: #1A1A1A مع شفافية 90%
- كل عنصر: ارتفاع 52px
  - أيقونة دائرية ملونة (36px) + اسم الحالة، نص أبيض
  - العناصر: متاح🟢 | بالخارج🟡 | مشغول🔴 | هاتف🔵 | طعام🟢 | نائم🟣 | سيارة⚫

## 🖥️ الشاشة 5: لوحة Admin / Super Admin
**الغرض:** إدارة الغرفة للمشرف — ما يملكه Guest + أدوات الإشراف

### الهيدر (مميز عن الهيدر العادي):
- نفس هيدر الغرفة + شارة "Admin" أو "Super Admin" بجانب الاسم
- لون الشارة: لون الرتبة

### الفرق عن شاشة Guest:
واجهة الشات نفسها تماماً مع إضافة واحدة فقط:

**النقر على اسم عضو في قائمة الحضور (يفتح Context Menu):**
- بطاقة منبثقة صغيرة، border-radius MD، ظل LG
- تحتوي فقط على الإجراءات المسموح بها:
  - **Admin يرى هذه الأزرار على Guest وMember فقط:**
    - 🔇 كتم المايك — WARNING
    - 🚪 طرد من الغرفة — DANGER
    - 🧹 مسح رسائله — TEXT_MUTED
  - **Super Admin يضيف:**
    - كتم Admin + طرده
    - ⚠️ تحذير رسمي — WARNING

**الإجراء المكتوم:** يظهر أيقونة 🔇 صغيرة حمراء فوق أفاتار الشخص في قائمة الأعضاء

## 🖥️ الشاشة 6: لوحة Master / Super Master
**الغرض:** إدارة متقدمة للغرفة — يملك كل ما يملكه Admin +

### إضافات على واجهة Admin:
**1. شريط إحصائيات الغرفة (شريط افقي أسفل الهيدر):**
- خلفية: BG_LIGHT، ارتفاع 36px
- 3 قيم: 👥 عدد المتواجدين | ⏱️ وقت التشغيل | 📊 الحد الأقصى
- FONT_SIZE_XS، TEXT_MUTED

**2. Context Menu الموسّع (عند النقر على عضو):**
- كل ما يملكه Admin + إضافة:
  - 🚫 حظر IP (24 ساعة) — DANGER
  - 👑 تعيين Admin (إذا كان Guest/Member)
  - 👑 إزالة Admin (إذا كان Admin)

**3. زر "أدوات Master" (⚙️ إضافي في هيدر الغرفة):**
- يفتح Bottom Sheet بالأدوات:
  - تفعيل/إيقاف كل الميكات دفعة واحدة
  - تفعيل/إيقاف كل الكاميرات
  - تعديل بانر ورسالة الترحيب (حقل نصي)
  - زر "حفظ التغييرات" — PRIMARY

**Super Master يضيف:**
- 🔒 حظر Device ID — DANGER (مع تأكيد)
- 💾 Backup الإعدادات — SUCCESS
- تعيين Master آخر ضمن الكوتة

**الغرض:** إدارة عالية المستوى — يملك كل ما يملكه Master +

**1. زر "إعدادات الغرفة" (منفصل عن ≡):**
يفتح شاشة كاملة تحتوي:

  - كل ثيم: معاينة مصغرة للنقش + اسم الثيم
  - المختار: حد 3px ACCENT + علامة ✓
- **قسم البانر:** حقل نصي متعدد الأسطر لرسالة الترحيب
- **إعدادات الغرفة:** السعة القصوى + تفعيل/إيقاف الكاميرا للجميع
- زر "حفظ" — PRIMARY

**2. Dual Machine Lock (Super Root فقط):**
في قائمة ≡ يضيف: "🔐 قفل الجهاز" → يفتح شاشة تعرض الأجهزة المرتبطة

## 🖥️ الشاشة 8: لوحة Owner Dashboard (owner.html)
**الغرض:** إدارة جميع غرف Owner — شاشة مستقلة كاملة

### الهيدر:
- خلفية: PRIMARY (#2C3E7A)
- عنوان: "لوحة تحكم الأونر" نص أبيض مركز
- أيقونة بيل 🔔 (يمين) للإشعارات
- زر 🔍 (يسار) للبحث

### شريط الإحصائيات السريعة:
- 4 بطاقات صغيرة متجاورة قابلة للتمرير:
  - 🏠 عدد الغرف (رقم كبير ACCENT)
  - 👥 إجمالي المستخدمين
  - 🟢 غرف نشطة

### 34. رؤية النظام البيومتري

ربط حساب كل مشرف (Admin فأعلى) ببصمة إصبعه أو بصمة وجهه على جهازه،
بحيث يصبح الدخول بالبصمة هو الأسلوب الأساسي الإلزامي — وليس كلمة المرور.

**المعيار المعتمد:** WebAuthn (W3C Standard) + FIDO2 — أعلى معيار أمان عالمي للمصادقة بدون كلمة مرور.

**المبدأ الذهبي:** بيانات البصمة أو الوجه لا تغادر الجهاز أبداً — فقط توقيع تشفيري يُرسل للسيرفر.

### 38. مسار الفشل والطوارئ

```
فشل البصمة 3 مرات متتالية:
━━━━━━━━━━━━━━━━━━━━━━━━━━
→ يطلب: كلمة المرور + رمز SMS/Email
→ بعد الدخول الناجح: خيار إعادة تسجيل البصمة

فقدان الجهاز:
━━━━━━━━━━━━━
→ تسجيل الدخول من جهاز جديد بكلمة المرور
→ النظام يكتشف: جهاز غير معروف لحساب مشرف
→ يطلب تأكيداً إضافياً: كود Email + سؤال أمان
→ يعرض: "هل تريد ربط هذا الجهاز الجديد ببصمتك؟"
→ إعداد بصمة جديدة على الجهاز الجديد
→ البصمة القديمة تُلغى تلقائياً (اختياري: يُبلَّغ المشرف)

إلغاء ربط البصمة (بواسطة Super Owner فقط):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
→ من لوحة Super Owner: يبحث عن المشرف
→ "إلغاء ربط البصمة" مع تأكيد مزدوج
→ يُرسل إشعار للمشرف
→ في الدخول التالي: يُطلب منه إعادة التسجيل
```

### 40. البنية التقنية (WebAuthn + FIDO2)

**المكتبة المختارة:** `@simplewebauthn/server` + `@simplewebauthn/browser`
**السبب:** أبسط وأشمل مكتبة WebAuthn للـ Node.js، مدعومة رسمياً.

```
التقنيات المستخدمة:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
السيرفر:   @simplewebauthn/server (Node.js)
الواجهة:   @simplewebauthn/browser (Vanilla JS)
التخزين:   جدول biometric_credentials في MySQL
التشفير:   ES256 / RS256 (مدعوم تلقائياً)
الأمان:    Challenge عشوائي لكل محاولة دخول
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**جداول قاعدة البيانات:**
```sql
-- بيانات بصمات المشرفين
CREATE TABLE biometric_credentials (
  id              VARCHAR(200) PRIMARY KEY,
  user_id         INT NOT NULL,
  credential_id   TEXT NOT NULL,        -- معرف البصمة
  public_key      TEXT NOT NULL,        -- المفتاح العام (Base64)
  device_name     VARCHAR(100),         -- "iPhone 15 - Safari"
  biometric_type  ENUM('fingerprint','face','both') DEFAULT 'fingerprint',
  counter         BIGINT DEFAULT 0,     -- حماية من Replay Attack
  is_active       BOOLEAN DEFAULT TRUE,
  registered_ip   VARCHAR(45),          -- IP عند التسجيل
  registered_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_used_at    TIMESTAMP,
  last_used_ip    VARCHAR(45),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- سجل محاولات البصمة (للمراقبة الأمنية)
CREATE TABLE biometric_attempts (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  user_id     INT NOT NULL,
  success     BOOLEAN NOT NULL,
  ip_address  VARCHAR(45),
  user_agent  VARCHAR(255),
  failure_reason VARCHAR(100),
  attempted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- الـ Challenges المؤقتة (تنتهي بعد 5 دقائق)
CREATE TABLE biometric_challenges (
  user_id     INT PRIMARY KEY,
  challenge   VARCHAR(500) NOT NULL,
  expires_at  TIMESTAMP NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

**API Endpoints الجديدة:**
```
POST /api/auth/biometric/register/start    ← يبدأ عملية التسجيل
POST /api/auth/biometric/register/finish   ← يكمل التسجيل
POST /api/auth/biometric/login/start       ← يبدأ عملية الدخول
POST /api/auth/biometric/login/finish      ← يتحقق ويمنح JWT
DELETE /api/auth/biometric/revoke          ← يلغي ربط البصمة (Super Owner)
GET  /api/auth/biometric/status            ← هل البصمة مسجلة؟
```
